Record Views

Perma.cc provides multiple record formats for each captured web page. You can navigate through the different formats using the tabs at the top of each page.

Capture View

Capture view is the default View for a Perma.cc record. This view displays the webpage, rendered with html, css, and javascript, with accompanying images. It best recreates the view and functionality as it existed when the Perma Link was created.

If you are in the Capture View, you’ll see an interactive archive of the cited web page as it appeared when the Perma Link was created. If the Perma Link is unvested, you will see the blue Perma icon overlay on the archived pages.

Screenshot View

The Screenshot View is a PNG image of the cited webpage as it existed when the Perma Link was created.

If you select the Screenshot View, you’ll see a screenshot of the original web page as it appeared when the Perma Link was created. If the Perma Link is unvested, you will see a blue Perma icon overlay on the image.

Live Page

If you select View Live Page, you’ll go directly to the original source URL. This allows you to check quickly to see if the original page is still live.

Record Status

Public Record

The default record status is Public. A Public record is a repository for publicly viewable materials archived using Perma.cc.

Most of the materials archived by Perma users are part of the Public Archive, which means the materials are publicly accessible to anyone who clicks the associated Perma Link or enters the URL for that Perma Link into their browser.

Dark Archive

A record with the Status of Dark Archive contains the captured url data, but is not publicly viewable.

Some of the materials archived by Perma users become part of the Dark Archive, which means they are preserved but not widely accessible. These materials remain available to the person who created the Perma Link. Once vested, these materials also are available to the Vesting User, Vesting Organization and Registrar responsible for the vesting decision.

Materials may become part of the Dark Archive for different reasons. The person who created or vested a Perma Link may designate it for the Dark Archive. Also, by default, Perma places in the Dark Archive any pages with a “noarchive” metatag or a Perma-specific exclusion in the robots.txt file. Finally, archived materials are placed in the Dark Archive upon receipt of a valid DMCA takedown notice.
